7 - File Permissions

Linux Users

User Accounts and the Root User

User permissions are assigned a user id (UID) which is a number used to track permissions.

For normal users, usually they are assigned the first avaliable user ID starting from 500.

The root user is assigned the UID of 0, and has complete permissions to do anything on the system.

System Accounts

Each process also has it’s own unique system account, which acts as its own user with its own set of permissions.
Generally UIDs under 500 are reserved for system accounts.

/etc/passwd

The /etc/passwd file contains information about permissions of user and system accounts.

        
        12345678910111213
        $ cat /etc/passwd
root:x:0:0:root:/root:/bin/bash
bin:x:1:1:bin:/bin:/sbin/nologin
daemon:x:2:2:daemon:/sbin:/sbin/nologin
adm:x:3:4:adm:/var/adm:/sbin/nologin
lp:x:4:7:lp:/var/spool/lpd:/sbin/nologin
sync:x:5:0:sync:/sbin:/bin/sync
shutdown:x:6:0:shutdown:/sbin:/sbin/shutdown
halt:x:7:0:halt:/sbin:/sbin/halt
mail:x:8:12:mail:/var/spool/mail:/sbin/nologin
news:x:9:13:news:/etc/news:
uucp:x:10:14:uucp:/var/spool/uucp:/sbin/nologin
...

This file contains a colon-separated list of information about user ids.
In the following order:

  • Username
  • Password
  • UID
  • Group ID (GID)
  • Comment (Description)
  • Location of the HOME directory for that user
  • Default shell

/etc/shadow

Because it poses a security risk to show passwords in the /passwd file as many programs can access it, passwords are put in a separate /etc/shadow file.
Only a few programs have access to this file.

Each account will have one record in this file, with something like:

        
        1
        rich:$1$.FfcK0ns$f1UgiyHQ25wrB/hykCn020:11627:0:99999:7:::

In the following order:

  • Login name (matches the passwd file)
  • Encrypted password
  • Date of last password change (date is in days since Jan 1, 1970)
  • Number of days before password must be changed
  • Days until password expiration
  • Days after password expires before the account is disabled
  • Date since the account was disabled (days since Jan 1, 1970)
  • Reserved Field (useless right now)

Creating, Modifying, and Removing Users

Adding a User

Users can be added with useradd.
useradd may not be in the PATH by default. If that’s the case, add the /usr/sbin/ directory.

If useradd is used without parameters, it uses the defaults. To see the defaults, use useradd -D or see the /etc/default/useradd file.

The defaults will have something similar to the following form:

        
        1234567
        GROUP=100
HOME=/home
INACTIVE=-1
EXPIRE=
SHELL=/bin/bash
SKEL=/etc/skel
CREATE_MAIL_SPOOL=yes
Parameter Meaning
GROUP Group ID
HOME Directory the new user folder is created in
INACTIVE -1 means the account won’t disable after password expiration
EXPIRE Date of expiration, here there is no expiration date
SHELL What shell to use (default bash in this case)
SKEL Contents of /etc/skel are copied to the new HOME directory
CREATE_MAIL_SPOOL Creates a file in mail directory

The /etc/skel will usually have files like .bashrc and .profile which is copied into the new HOME directory for that user.

To add parameters to the useradd command, you would use flags. Adding a -D flag before the other flags will mean a change to the defaults.

For example:

        
        1
        $ sudo useradd -D -s /bin/tsch

will override the default bash shell and use the tsch shell instead for that user.

Removing a User

userdel can be used to remove a user from the passwd file.
The user’s HOME directory will not be touched unless you use the -r parameter.

        
        1
        $ userdel -r testUser

Modifying a User

There are many commands to modify a user.

usermod

usermod has the same flags as useradd but can be used to change existing users.
Some extras include -l for changing the login name, -p for the password, and so on.

passwd and chpasswd

passwd can be used the change a user’s password. You can only change your own with this, but the root user can change any user’s.

        
        1234
        $ passwd testUser
New UNIX password:
Retype new UNIX password:
passwd: all authentication tokens updated successfully.

chpasswd can be used for batch changes. It takes a file input containing lines username:new_password colon-separated pairs.

        
        1
        $ chpasswd < batchChanges.txt

chsh chfn and chage

These are specialized commands that do specific changes that usermod can also do.

chsh (change shell) is used for changing a user’s shell.

        
        1
        $ chsh -s [fullpath to shell] testUser

chfn (change finger) let you set the comment of a user using a template.

        
        123456
        $ chfn testUser
Changing finger information for testUser.
Name []: Ima Test
Office []: Director of Technology
Office Phone []: (123)555-1234
Home Phone []: (123)555-9876

To view this comment for users, use finger:

        
        1
        $ finger testUser

finger is usually disabled due to security reasons however.

Finally chage (change aging) changes information about user expiration and inactivity.


Linux Groups

Users are part of groups with a unique ID (GID), and share permissions.
Similar to users, group information is stored in the /etc/group file.

        
        123456
        root:x:0:root
bin:x:1:root,bin,daemon
daemon:x:2:root,bin,daemon
sys:x:3:root,bin,adm
adm:x:4:root,adm,daemon
...

There are 4 fields:

  • Name
  • Password
  • GID
  • Users belonging to the group

Adding New Groups

To add a group, use the groupadd command (found in /usr/sbin)

To add members to a group, use usermod

        
        12
        $ /usr/sbin/groupadd testGroup
$ /usr/sbin/usermod -G testGroup testUser

The -G flag appends the user to the new group rather than replacing it.

Modifying Groups

Using the groupmod command, the group name and GID can be changed using flags.

        
        1
        $ groupmod -n testGroup test_group

will change the name from testGroup to test_group.


File Permissions

This has been lightly touched in 0 - Basic Unix Commands.

When using ls -l, we will see all the permissions attached to a file.

        
        12
        $ ls -l
drwxr-xr-x 1 testUser testGroup 197609   0 Mar  1 16:44 testDir

To decode the symbols:

Object Type Owner Permissions Group Permissions General Permissions Owner Name Group Name
d - directory rwx - (Read, Write, Execute) r-x (Read, Execute) r-x (Read, Execute) testUser testGroup

Other object types include:

  • ‘-’ file
  • ‘d’ directory
  • ‘l’ link
  • ‘c’ character device
  • ‘b’ block device
  • ‘n’ network device

Default Permissions

When a new file is created, it takes the default permissions.
To view it, use umask

        
        12
        $ umask
0022

This is an octal number which converts to binary for the permissions.

By default, the permission of a file is 666 for a file, and 777 for a directory.
The mask is subtracted from this value to get the final permission number:

For a file:

        
        1234
         0666
-0022
 ====
 0644

What does this number mean?
Each octal place represents a permission group:

Number System Sticky Bit Owner Group Everyone
Octal 0 6 4 4
Binary 0 110 100 100
Permissions 0 rw- r- - r- -

Ignore the sticky bit for now.

To change the umask, simply put a new number in the umask.

        
        123
        $ umask 026'
$ umask
0026

Changing Permissions

chmod can be used the change file permissions.

        
        1
        $ chmod [options] permissions filename

The permissions can be set using an octal number (the value is used as is):

        
        123
        $ chmod 760 testFile
$ ls -l
-rwxrw---- 1 testUser testGroup 197609   0 Mar  1 16:24 testFile

It can also be set symbollically with the approach:

        
        1
        [ugoa...][+-=][rwx...]

In the first set, we have:

  • u: user
  • g: group
  • o: other
  • a: all

The second set, we have:

  • +: add permission
  • -: remove permission
  • =: set permission to

The third set, we have the basics along with other new parameters:

  • r: read
  • w: write
  • x: execute
  • X: execute only if it is a directory
  • u: permissions of owner
  • g: permissions of group
  • o: permissions of other

For example:

        
        12345678
        $ ls -l
-rwxrw---- 1 testUser testGroup 197609   0 Mar  1 16:24 testFile
$ chmod o+rwX testFile
$ ls -l
-rwxrw-rw- 1 testUser testGroup 197609   0 Mar  1 16:24 testFile
$ chmod u-x testFile
$ ls -l
-rw-rw-rw- 1 testUser testGroup 197609   0 Mar  1 16:24 testFile

Changing Ownership

To change the owner of a file, use chown (change owner).
To change the group of a file, use chgrp (change group).

        
        1
        chown [options] owner[.group] file

Note the chown command can also change the group if attached to the owner using a period.

        
        1234567891011121314
        $ ls -l
-rw-rw-rw- 1 testUser testGroup 197609   0 Mar  1 16:24 testFile

$ chown someUser.someGroup testFile
$ ls -l
-rw-rw-rw- 1 someUser someGroup 197609   0 Mar  1 16:24 testFile

$ chown someOtherUser testFile
$ ls -l
-rw-rw-rw- 1 someOtherUser someGroup 197609   0 Mar  1 16:24 testFile

$ chgrp someOtherGroup testFile
$ ls -l
-rw-rw-rw- 1 someOtherUser someOtherGroup 197609   0 Mar  1 16:24 testFile

If the group name is the same as the user, simply using the period will work:

        
        1
        $ chown someUser. testFile
